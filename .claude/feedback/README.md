# 自动优化反馈机制

## 概述

本机制确保在开发过程中学习到的经验自动反馈到 SKILL 和文档中，形成持续改进的闭环。

## 架构设计

```
┌─────────────────────────────────────────────────────────────────┐
│                        开发会话                                   │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐        │
│  │ 问题修复  │  │ 新模式   │  │ 最佳实践  │  │架构设计  │        │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘  └────┬─────┘        │
└───────┼─────────────┼─────────────┼─────────────┼────────────────┘
        │             │             │             │
        ▼             ▼             ▼             ▼
┌─────────────────────────────────────────────────────────────────┐
│                    feedback 触发检测                              │
│  • 错误修复模式匹配                                               │
│  • 新发现的配置/路径                                              │
│  • 重复出现的问题                                                 │
│  • 用户明确要求记住的内容                                          │
│  • 架构设计理念抽象                                               │
└─────────────────────────────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────────────────────────────┐
│                    自动更新目标                                    │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐          │
│  │ MEMORY.md    │  │ SKILL 文件   │  │ CLAUDE.md    │          │
│  │ (会话记忆)   │  │ (技能改进)   │  │ (项目规范)   │          │
│  └──────────────┘  └──────────────┘  └──────────────┘          │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │              patterns.yaml (架构模式库)                   │   │
│  └──────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

## 触发条件

### 自动触发
1. **错误修复**: 当同一个错误被修复超过 2 次
2. **新模式**: 发现新的路径、配置、工具用法
3. **最佳实践**: 验证有效的解决方案模式
4. **架构设计**: 发现新的架构模式或数据流

### 手动触发
用户明确说 "记住这个" 或 "更新文档"

## 反馈目标

### 1. MEMORY.md (短期记忆)
记录会话中发现的临时信息：
- 当前项目的特定路径
- 临时解决方案
- 待验证的假设

### 2. SKILL 文件 (技能知识)
需要更新的技能文件：
- `dev`: 开发命令相关
- `fix-*`: 错误修复规则
- `openresty-lua-plugins`: OpenResty 开发模式
- `web-admin-frontend`: 前端架构设计

### 3. CLAUDE.md (项目规范)
项目级别的永久规范：
- 目录结构
- 编码规范
- 工作流程

### 4. patterns.yaml (架构模式库)
可复用的架构设计模式：
- 数据流设计
- 组件设计模式
- 配置模型

## 架构设计理念自动抽象

### 检测规则

| 触发条件 | 检测方式 | 更新目标 |
|----------|----------|----------|
| 新增目录结构 | 目录深度 >= 3 或新文件类型 | SKILL 目录结构 |
| 新增数据流 | 新 API 端点或转换步骤 | SKILL 数据流设计 |
| 新增组件模式 | 新 Vue 组件或 Lua 模块 | SKILL 组件设计 |
| 新增配置模型 | 新 JSON 字段或 nginx 指令 | SKILL 配置模型 |

### 抽象流程

```
开发完成新功能
      │
      ▼
检测是否为架构变更
      │
      ├── 是 → 提取设计理念
      │              │
      │              ▼
      │         匹配已有模式
      │              │
      │              ├── 已存在 → 跳过
      │              │
      │              └── 新模式 → 提示更新 SKILL
      │
      └── 否 → 跳过
```

### 提示格式

```
[feedback] 检测到架构设计变更：

新增内容:
  - 目录: src/loadbalance/
  - 数据流: JSON → generator.lua → nginx.conf
  - API: POST /api/deploy/apply

建议更新:
  1. MEMORY.md - 记录新目录路径
  2. web-admin-frontend SKILL - 更新数据流设计
  3. patterns.yaml - 添加部署流程模式

是否执行更新? [Y/n/a(全部)]
```

## 实现文件

| 文件 | 用途 |
|------|------|
| `.claude/skills/feedback` | 反馈机制主技能 |
| `.claude/projects/*/memory/` | 项目记忆存储 |
| `.claude/feedback/patterns.yaml` | 可复用模式库 |
| `.claude/feedback/updates.json` | 待更新队列 |

## 使用方式

### 自动反馈
在会话结束时，系统自动检测并提示更新：

```
[feedback] 检测到以下可更新内容：
  - 路径: /home/mxp/Skills/build/openresty -> 修正为 build/openresty
  - 修复: nginx 路由优先级 (location /api/ 需在 / 之前)
  - 架构: 新增配置驱动架构模式
是否更新 MEMORY.md? [Y/n]
```

### 手动触发
```
/feedback              # 查看当前会话的可优化内容
/feedback apply        # 应用所有建议的更新
/feedback architecture # 查看架构变更摘要
```

## 规则示例

### 错误修复规则 (tools/fixers/runtime/nginx-routing.yaml)
```yaml
id: nginx-location-order
pattern:
  - "proxy_pass.*vite"
  - "api.*504|404"
fix: |
  确保 location /api/ 在 location / 之前定义。
  Nginx 按最长匹配优先，但相同前缀时按定义顺序。
files:
  - "**/nginx.conf"
```

### 架构模式 (patterns.yaml)
```yaml
architecture_patterns:
  config_driven:
    name: "配置驱动架构"
    description: "JSON 配置 → 配置生成器 → nginx.conf"
    data_flow:
      - "用户修改 JSON"
      - "API 保存配置"
      - "生成器转换格式"
      - "控制器验证重载"
```

## 维护原则

1. **最小化原则**: 只记录真正有价值的信息
2. **验证原则**: 确保信息准确后再记录
3. **去重原则**: 检查已有内容避免重复
4. **时效原则**: 定期清理过时信息
5. **抽象原则**: 将具体实现抽象为可复用模式
