# Feedback Skill

自动优化反馈机制，在开发过程中学习并更新项目知识库。

## 触发条件

- 用户说 "记住这个"、"更新文档"、"反馈到 skill"
- 会话中修复了重复出现的错误
- 发现新的最佳实践或架构设计模式
- 用户调用 `/feedback` 命令

## 反馈目标

| 目标 | 内容 | 更新频率 |
|------|------|----------|
| MEMORY.md | 会话记忆、临时发现 | 每次会话 |
| SKILL 文件 | 技能知识、修复规则 | 发现新模式时 |
| CLAUDE.md | 项目规范、工作流程 | 重大变更时 |
| patterns.yaml | 可复用模式库 | 验证有效后 |

## 使用方式

| 命令 | 说明 |
|------|------|
| `/feedback` | 查看当前会话的可优化建议 |
| `/feedback apply` | 应用所有建议的更新 |
| `/feedback status` | 查看反馈系统状态 |
| `/feedback architecture` | 查看架构变更摘要 |

## 更新规则

### 路径配置
当发现路径变化时更新 `tools/scripts/common.sh`、`tools/bin/dev`、MEMORY.md。

### 错误修复规则
当同一错误修复 2 次以上时，在 `tools/fixers/` 下创建或更新规则文件。

### 架构模式
当发现新架构模式时，抽象为可复用模式，更新到对应 SKILL 和 patterns.yaml。

## 架构变更检测

检测到以下变更时自动提示更新：
- 新增目录深度 >= 3 或新增文件类型
- 新增 API 端点或数据转换步骤
- 新增 Vue 组件或 Lua 模块
- 新增 JSON 字段或 nginx 指令生成

## 输出格式

```
[feedback] 更新 <target>: <description>

变更内容:
- 添加/修改: <具体内容>
- 原因: <为什么需要这个变更>
- 影响: <影响范围>
```
