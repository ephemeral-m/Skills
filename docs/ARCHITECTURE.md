## 1. 项目概述

### 1.1 项目定位

Skills 是一个 **OpenResty 开发技能库**，旨在通过 AI 辅助开发的方式，提升 Nginx/Lua 插件的开发效率。项目整合了：

- **OpenResty 运行时环境**：完整的 OpenResty 1.29.2.1 源码及第三方模块
- **Lua 插件开发框架**：标准化的 HTTP/Stream 插件开发模式
- **Web 管理界面**：Vue 3 前端 + OpenResty 后端的配置管理系统
- **AI 开发助手**：通过 Claude Code Skills 实现 AI 辅助开发

### 1.2 核心价值

```
┌─────────────────────────────────────────────────────────────┐
│                    Skills 核心价值                           │
├─────────────────────────────────────────────────────────────┤
│  1. 跨平台开发：Windows 编辑 + Linux 运行                    │
│  2. AI 辅助开发：Skills 体系实现零 Token 消耗的自动化        │
│  3. 标准化流程：统一的构建、测试、部署流水线                  │
│  4. 可视化管理：Web 界面管理 Nginx 配置                      │
│  5. 知识积累：项目记忆 + 反馈机制持续学习                     │
└─────────────────────────────────────────────────────────────┘
```

## 2. 设计理念

### 2.1 跨平台开发模式

**核心理念：Windows 开发 + Linux 远程运行**

```
┌─────────────────────────────────────────────────────────────────┐
│                    Windows 本地环境                              │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  • 代码编辑 (IDE/编辑器)                                      ││
│  │  • Git 版本管理                                               ││
│  │  • Python dev CLI (通过 SSH 控制远程)                         ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
                              │
                        SSH (paramiko)
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    Linux 远程环境                                │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  • OpenResty 构建和运行                                       ││
│  │  • 所有 Shell 脚本执行                                        ││
│  │  • 服务启动/停止/测试                                          ││
│  │  • 生产流量转发                                                ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
```

**设计考量：**

| 问题 | 解决方案 |
|------|----------|
| Windows 无法运行 Shell 脚本 | 通过 SSH 远程执行 |
| OpenResty 只能在 Linux 构建 | 构建过程全部在远程 |
| 代码同步问题 | dev CLI 提供 sync 命令 |
| 路径差异 | 使用相对路径，运行时转换 |
| 编码差异 | UTF-8 统一处理 |

### 2.2 零 Token 消耗原则

Skills 设计遵循 **零 Token 消耗** 原则：

```
┌─────────────────────────────────────────────────────────────────┐
│                    Token 消耗对比                                │
├─────────────────────────────────────────────────────────────────┤
│  传统方式：                                                      │
│  用户请求 → AI 读取大量文件 → 生成代码 → 消耗大量 Token          │
│                                                                  │
│  Skills 方式：                                                   │
│  用户请求 → Skills 直接执行脚本 → 返回结果 → Token 消耗 ≈ 0      │
└─────────────────────────────────────────────────────────────────┘
```

**实现方式：**

1. **Slash Command**: `/dev build` 直接执行构建，无需 AI 理解
2. **渐进式加载**: 错误规则按需加载，不预加载全部
3. **模板化生成**: 插件生成基于模板，减少推理开销

```bash
# 配置
./configure --prefix=/path/to/build/openresty \
    --with-http_ssl_module \
    --with-stream \
    --with-stream_ssl_module

# 编译
make -j$(nproc)

# 安装
make install
```

### 5.2 核心 Skills 说明

#### 5.2.1 dev Skill

**用途**: 统一开发命令入口

**命令**:

| 命令 | 说明 |
|------|------|
| `/dev all` | 完整流水线 (sync → build → start → test) |
| `/dev build` | 编译构建 |
| `/dev test --dt` | 运行测试 |
| `/dev start` | 启动服务 |
| `/dev stop` | 停止服务 |
| `/dev sync` | 同步代码 |

**流水线**:

```
sync → build → start → test
  │       │       │      │
  └───────┴───────┴──────┴── 失败时自动分析错误
```

#### 5.2.2 openresty-lua-plugins Skill

**用途**: 生成 HTTP/Stream Lua 插件

**工作流程**:

```
1. 需求评估 → 能否用 Nginx 原生配置？
2. 收集需求 → 插件名称、类型、功能描述
3. 定义格式 → 配置项、数据格式
4. 生成产物 → Lua 文件、文档、测试用例
```

**设计原则**:

1. **优先 Nginx 原生配置**: 仅在无法满足时开发 Lua 插件
2. **禁止高风险行为**: 阻塞 I/O、全局变量写入、无限循环
3. **标准化模板**: 统一的插件结构和编码规范

**错误修复系列**:

| Skill | 触发场景 | 错误规则文件 |

| fix-test | 测试失败 | `tools/fixers/test/*.yaml` |
